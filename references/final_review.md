# 软著材料最终复核清单

在宣称"软著材料完成"前，必须逐项检查此清单。

---

## 1. 交付物完整性

- [ ] 目标目录包含三件套：申请表、用户手册、源代码文档
- [ ] 文件名包含用户确认的"软件全称 + 版本号"
- [ ] 文件总数与预期一致，无遗漏

---

## 2. 一致性检查

- [ ] 三份文档中**软件全称**完全一致
- [ ] 三份文档中**版本号**完全一致
- [ ] 三份文档中**著作权人**完全一致
- [ ] 申请表"主要功能和技术特点"与用户手册章节范围一致
- [ ] 开发完成日期、首发表、发表城市等关键日期口径一致
- [ ] 用户要求删除的模块词在所有材料中无残留

---

## 3. 申请表检查

- [ ] 必填字段均已填写（软件全称、版本号、著作权人、开发完成日期等）
- [ ] 用户要求留空的字段保持空白
- [ ] 权利范围、权利取得方式与用户确认值一致
- [ ] 无旧项目名称或无关内容的残留

---

## 4. 用户手册检查

- [ ] 结构完整：封面 → 引言 → 系统要求 → 安装启动 → 功能说明 → FAQ → 联系方式
- [ ] 章节按真实系统菜单或业务流程展开
- [ ] 每个核心功能：有入口说明、操作步骤、结果反馈
- [ ] 无 AI 味、营销口号、模板堆砌
- [ ] 截图来自真实系统页面或用户提供的文件（非生成图片）
- [ ] 暂不截图时，手册中有清晰可见的截图占位文字
- [ ] 术语表术语与正文一致

---

## 5. 源代码检查

- [ ] 代码全部来自真实项目源码（非编造）
- [ ] 排除目录已正确排除（node_modules、dist、构建产物、依赖包、第三方库、测试固件）
- [ ] 页数 >=60 时取前30页 + 后30页；不足60页提交全部
- [ ] 每页行数 >=50 行（不含大量空行填充）
- [ ] 前后端分类正确：前端放入口/路由/页面/组件，后端放Controller/Service/Model
- [ ] 页眉/页脚包含软件全称和版本号
- [ ] 页码连续，格式统一
- [ ] 已清理：尾随空格、连续多空行、密钥/token/密码
- [ ] 保留正常缩进和必要注释

---

## 6. 格式质量

- [ ] 页边距：上下2cm，左3cm，右2cm（A4）
- [ ] 标题字体：宋体 16pt 加粗
- [ ] 正文字体：宋体 12pt，1.5倍行距
- [ ] 代码字体：等宽 8-9pt
- [ ] 页码格式："第 X 页"
- [ ] 表格不越界，截图不压字

---

## 7. 最终检查

- [ ] 已检查用户目标目录中的最终文件，而非中间产物
- [ ] 文件可正常打开，内容可读
- [ ] 用户确认的禁用词在全部文件中已清理
- [ ] 三件套资料可整体打包交付
