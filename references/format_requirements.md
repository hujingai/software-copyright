# 软著申请格式要求

参考中国版权保护中心（CPCC）软件著作权登记材料格式规范。

---

## 源代码文档格式

| 项目 | 要求 |
|------|------|
| 纸张 | A4 (210mm × 297mm) |
| 页边距 | 上 2cm、下 2cm、左 3cm、右 2cm |
| 行数 | 每页 ≥ 50 行（默认50行，建议不超过60行） |
| 页码 | 底部居中，"第 X 页" 格式 |
| 页眉 | 软件全称 + 版本号 + "源代码文档" |
| 代码字体 | 等宽字体，8-9pt |
| 排版 | 保留正常缩进，清理尾随空格和连续多空行 |
| 页数策略 | ≥60页取前30+后30页；<60页提交全部 |
| 前端优先 | 入口文件 → 路由配置 → 页面组件 → 接口封装 → 公共组件 |
| 后端优先 | 入口 → Controller → Service → Repository/DAO → Model/Entity → 工具类 |

---

## 用户手册格式

| 项目 | 要求 |
|------|------|
| 封面字体 | 宋体 16pt 加粗（软件名称）、14pt（"用户使用说明书"） |
| 正文标题 | 宋体 14-16pt 加粗 |
| 正文字体 | 宋体 12pt |
| 行距 | 1.5 倍 |
| 页边距 | 上 2.5cm、下 2cm、左 3cm、右 2cm |
| 页面大小 | A4 |
| 章节编号 | 一、二、三…… / 1.1、1.2…… |
| 图片 | 居中放置，图注 10pt 居中 |

---

## 申请表格式

| 项目 | 要求 |
|------|------|
| 页面大小 | A4 |
| 格式 | 表格形式（CPCC 标准格式） |
| 字体 | 宋体 10-12pt |
| 必填字段 | 软件全称、版本号、著作权人、开发完成日期 |
| 留空字段 | 按用户要求保持空白 |
| 安全清理 | 不包含旧项目名称、无关信息 |

---

## 文件命名规范

```
[软件全称]_[文档类型]_V[版本号].pdf
```

示例：
- `任务管理系统_源代码文档_V1.0.pdf`
- `任务管理系统_用户手册_V1.0.pdf`
- `任务管理系统_申请表_V1.0.pdf`
